## Problem Statement

用户完成一节教学模拟课后，无法回顾已完成的课堂。当前前端 LandingPage 有一个 disabled 的"历史"按钮，但没有对应的功能实现。需要构建一个课堂历史记录系统，让用户可以浏览已完成的会话列表并查看每节课的详细消息时间线。

## What Makes This Cool

一节AI模拟课的数据已经完整存储在数据库中（消息、检查点进度、学生表现）。这个功能是把已有数据变成用户可感知的价值。点击一个卡片就能"回放"整节课的对话过程，对演示场景非常有说服力。

## Constraints

- 前端使用 **rough-design-skill** 设计风格（手绘/hand-drawn 美学），参考 `.claude/skills/rough-design-skill/SKILL.md`
- 核心视觉元素：粗边框 `3px solid #1A1A1A`、硬阴影 `4px 4px 0px 0px #1A1A1A`、轻微旋转 `rotate(-0.5deg)`、胶带装饰
- 字体：标题用 Plus Jakarta Sans，正文用 Be Vietnam Pro（已有本地字体文件）
- 配色：sticky-note 色系（`#E3F2FD` 蓝、`#E8F5E9` 绿、`#FFF9C4` 黄、`#FCE4EC` 粉）+ paper 白 `#fafafa`
- 前端组件遵循 ONE Wrapper 模式（每个文件一个 Wrapper styled component）
- 后端遵循 `models/` 目录下的 feature-based 模块结构
- 数据已存在数据库中，不需要新建表，只需要新增查询 API
- 优先使用 TDD 流程开发

### 前端设计规范（rough-design-skill）

实现时必须遵循以下 rough design 规则：

| 元素 | 规范 | CSS |
|------|------|-----|
| 边框 | 粗、硬 | `border: 3px solid #1A1A1A` |
| 阴影 | 硬、偏移 | `box-shadow: 4px 4px 0px 0px #1A1A1A` |
| 不完美感 | 轻微旋转 | `transform: rotate(-0.5deg)` |
| 胶带 | 半透明 | `background: rgba(227,242,253,0.6)` |
| Hover | 阴影远离 | `box-shadow: 8px 8px 0px 0px #1A1A1A; transform: translate(-4px, -4px)` |
| 按钮 | 按压反馈 | `&:active { transform: scale(0.95) }` |

**禁止事项：**
- 不用软阴影（`rgba(0,0,0,0.1)` blur）
- 不用细边框（< 3px）
- 不用完美对齐（需要轻微旋转）
- 不用圆角（或极少使用）
- 不用 Inter/Roboto/Arial 字体

## Prerequisites

**会话生命周期数据缺失问题（必须在历史功能之前修复）:**

当前代码中存在两个数据问题，如果不修复，历史页面将永远为空：

1. **`end_time` 从未被写入。** `TeachingSessionModel.end_time` 列存在，但没有代码在会话结束时设置它。
2. **`status` 从未被更新。** 所有会话的 `status` 始终为 `"running"`，没有代码将其转为 `"completed"` 或 `"interrupted"`。

**修复方案:** 在 `SessionOrchestrator`（观察模式）和 `TeacherSessionController`（教师模式）的会话结束流程中，添加会话生命周期管理：
- 会话正常结束时: 设置 `status = "completed"`, `end_time = now()`, `duration_seconds = 实际时长`
- 会话异常中断时: 设置 `status = "interrupted"`, `end_time = now()`, `duration_seconds = 实际时长`

这个修复应作为历史功能的 Task 0，用 TDD 流程实现。

## Premises

1. **不需要新建数据库表** — `teaching_sessions`、`messages`、`checkpoint_plans` 三张表已经包含所有需要的数据，只需新增查询接口。
2. **显示所有会话，包括 running** — 由于 `status` 当前始终为 `"running"`，列表 API 不过滤状态。前端对 running 状态的会话显示"进行中"标签，对 completed 显示"已完成"。Prerequisites 修复后，running 的会话数量会自然减少。
3. **消息时间线按时间正序显示** — 最早的在最上面，模拟课堂的自然流程。
4. **不需要删除/编辑功能** — 这是只读的回顾功能。
5. **`students_config` 是 JSON 数组** — 存储格式为 `[{name, level, ...}, ...]`，学生数量用 `len(students_config)` 获取。
6. **无检查点计划的会话需要处理** — 有些会话可能没有 `checkpoint_plans` 记录（创建后未开始教学就中断），列表 API 需要左连接并返回 `checkpoint_progress: null`。

## Approaches Considered

### Approach A: Session list + detail view（推荐）

**Summary:** 两个页面 — 列表页展示所有历史会话卡片，点击进入详情页查看消息时间线和检查点进度。

**Effort:** S (human: ~1 day / CC: ~30 min)

**Risk:** Low

**Pros:**
- 最小实现，复用已有 ORM 模型和 API 模式
- 前端只需 2 个新页面 + 1 个新 API（详情复用已有接口）
- 数据已存在，零迁移成本

**Cons:**
- 无搜索/过滤功能（demo 场景下数据量不大，不构成问题）

**Reuses:**
- `TeachingSessionModel`、`MessageModel`、`CheckpointPlanModel` 已有 ORM
- `getSessionMessages()` 已有前端 API（`apis/session.ts`）
- `getCheckpointPlan()` 已有前端 API（`apis/observation.ts`）
- LandingPage 的 sketch 设计风格

### Approach B: 单页折叠面板

**Summary:** 在一个页面内，用可折叠的手风琴组件展示每个会话。展开后直接显示消息。

**Effort:** S

**Risk:** Med

**Pros:**
- 不需要新增路由
- 交互更紧凑

**Cons:**
- 会话多时页面很长，体验差
- 无法分享单个会话的 URL
- 消息量大的会话展开后性能差

### Approach C: 带搜索和分析的完整 dashboard

**Summary:** 列表 + 详情 + 搜索过滤 + 学生表现统计图表。

**Effort:** L

**Risk:** Med

**Pros:**
- 功能最完整

**Cons:**
- 大量额外工作（图表库、统计聚合、过滤 UI）
- 延迟交付，不符合 demo/completion 目标

## Recommended Approach

**Approach A: Session list + detail view.** 最小可行方案，数据已在数据库中，只需新增 1 个 API 和 2 个页面。对 demo 场景来说够用，后续可以逐步加搜索和统计。

## Architecture

### Backend 新增

#### 0. 会话生命周期管理（前置任务）

**修改文件:**
- `models/session/services/observation_service.py` — SessionOrchestrator 结束时更新 session
- `models/session/services/teacher_service.py` — TeacherSessionController 结束时更新 session（如果已有结束流程）

**实现:**
```python
async def _finalize_session(self, db: AsyncSession, status: str):
    """更新会话的结束状态"""
    self.session.status = status  # "completed" 或 "interrupted"
    self.session.end_time = datetime.now(ZoneInfo("Asia/Shanghai"))
    if self.session.start_time and self.session.end_time:
        self.session.duration_seconds = int(
            (self.session.end_time - self.session.start_time).total_seconds()
        )
```

#### 1. 会话列表 API

**文件:** `backend/models/session/router.py`（在现有文件中新增路由）

```
GET /sessions/
```

返回所有会话列表，按 `start_time` 倒序。

**Response:**
```json
[
  {
    "id": 1,
    "topic": "Python 变量与数据类型",
    "teaching_mode": "heuristic",
    "status": "completed",
    "start_time": "2026-04-12T14:30:00+08:00",
    "end_time": "2026-04-12T15:15:00+08:00",
    "duration_seconds": 2700,
    "student_count": 8,
    "checkpoint_progress": {
      "total": 5,
      "completed": 4,
      "current_index": 3
    }
  },
  {
    "id": 2,
    "topic": "函数基础",
    "teaching_mode": "didactic",
    "status": "running",
    "start_time": "2026-04-14T10:00:00+08:00",
    "end_time": null,
    "duration_seconds": null,
    "student_count": 6,
    "checkpoint_progress": null
  }
]
```

**实现要点:**
- 查询 `TeachingSessionModel`，LEFT JOIN `CheckpointPlanModel`
- `student_count`: `len(session.students_config)`（`students_config` 是 JSON 数组）
- `checkpoint_progress`: 从 `plan_data` JSON 的 `checkpoints` 数组计算 `total`（长度）和 `completed`（`state == "complete"` 的数量），`current_index` 直接取 `plan_data["current_index"]`
- 无 checkpoint_plans 记录的会话返回 `checkpoint_progress: null`
- 按 `start_time DESC` 排序
- 单次查询 + 关联加载，避免 N+1

#### 2. 会话详情 API

已有接口可直接复用，无需新增:
- `GET /sessions/{session_id}/messages` → 消息时间线（`apis/session.ts` 中的 `getSessionMessages`）
- `GET /checkpoint-plans/{session_id}` → 检查点计划（`apis/observation.ts` 中的 `getCheckpointPlan`）

前端详情页组合调用这两个接口即可。

### Frontend 新增

#### 1. SessionHistory 页面（列表页）

**文件:** `frontend/src/views/SessionHistory.tsx`

**布局:**
- 顶部导航栏（与 LandingPage 一致的 `top-nav` 样式：粗底边框、硬阴影、SimuSketch 品牌）

**会话卡片（rough-design-skill 风格）:**
- 背景: sticky-note 色系（根据 teaching_mode 区分颜色）
  - 灌输式: `#FFF9C4`（黄色便签）
  - 启发式: `#E3F2FD`（蓝色便签）
  - 讨论式: `#E8F5E9`（绿色便签）
- 边框: `3px solid #1A1A1A`
- 阴影: `4px 4px 0px 0px #1A1A1A`
- 轻微旋转: 交替 `rotate(-0.5deg)` / `rotate(0.5deg)`
- Hover: 阴影扩大 + 平移（标准 rough hover 动效）
- 可选: 卡片顶部添加胶带装饰（`tape` 元素）

**卡片内容:**
- topic（标题，Plus Jakarta Sans 粗体）
- teaching_mode 标签 + status 标签（手绘风格小标签，粗边框，`padding: 2px 8px`）
- 日期和时长（Be Vietnam Pro 正文）
- 检查点进度条（手绘风格：虚线边框容器 + 实色填充条，不完美边缘用 `clip-path`）

**teaching_mode 中文映射:**
| 值 | 显示 | 卡片背景色 |

## Success Criteria

- 会话结束时 `status`/`end_time`/`duration_seconds` 正确更新
- LandingPage 的"历史"按钮可点击，导航到历史记录页
- 历史记录页展示所有会话卡片（包括进行中、已完成、已中断）
- 点击卡片进入详情页，左侧显示消息时间线（正序），右侧显示检查点进度
- 无检查点计划的会话在列表中正常显示（不崩溃）
- 空状态（无历史记录）有友好的提示 UI
- 后端测试覆盖列表 API 的正常路径、空数据路径、无检查点计划路径
