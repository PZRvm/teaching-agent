## 页面描述

教师模式的会话配置页面，用户在此输入教学主题、配置学生信息，系统将自动生成教学检查点计划。与观察模式配置类似，但不需要选择教学模式（因为教师模式由用户手动控制教学流程）。

## 组件列表

### 1. 导航栏
- **位置**：页面顶部
- **内容**：
  - 左侧：返回按钮（← 返回）
  - 中间：页面标题"教师模式 - 会话配置"
- **样式**：
  - 背景：白色半透明
  - 底部边框：2px 手绘虚线

### 2. 步骤 1：教学主题输入
- **布局**：卡片容器，内边距 24px
- **背景**：绿色便签色 `var(--sticky-green)`（教师模式使用绿色主题）
- **组件**：
  - 标题："步骤 1：教学主题"（24px 粗体）
  - 提示文字："请输入本次教学的主题内容"
  - 输入框：
    - 宽度：100%
    - 占位符："例如：Python函数的定义与调用"
    - 手绘边框输入框样式
    - 支持多行输入（textarea）

### 3. 步骤 2：学生配置
- **布局**：卡片容器，内边距 24px
- **背景**：绿色便签色
- **组件**：与观察模式配置完全一致

#### 学生创建方式切换
- 三个切换按钮：
  - "手动添加"（默认选中）
  - "随机生成"
  - "JSON导入"
- 样式：小按钮，选中时绿色高亮

#### 手动添加模式
- **学生列表**：
  - 显示已添加的学生卡片
  - 每个学生显示：姓名、等级标签(A/B/C)
  - 每个学生旁边有删除按钮

- **添加学生表单**：
  - 姓名：输入框
  - 等级：单选（A-优秀 / B-中等 / C-基础）
  - 态度：单选（积极 / 中立 / 被动）
  - 学习能力：滑块（1-10）
  - 添加按钮："[+ 添加学生]"

#### 随机生成模式
- **配置项**：
  - 班级人数：数字输入框（默认 10）
  - 优秀比例：滑块（10%-50%，默认 20%）
  - 中等比例：滑块（30%-70%，默认 60%）
  - 基础比例：自动计算
  - 生成按钮："[🎲 随机生成班级]"

- **生成结果**：
  - 显示生成的学生列表（只读）
  - 支持"重新生成"

#### JSON导入模式
- **配置项**：
  - 文本域：用于粘贴 JSON 格式的学生数据
  - 占位符：显示 JSON 示例
  - 导入按钮："[📥 导入数据]"
  - 验证提示：显示导入是否成功

### 4. 步骤 3：检查点生成提示
- **布局**：卡片容器，内边距 24px
- **背景**：绿色便签色
- **组件**：
  - 图标：💡（48px）
  - 标题："步骤 3：检查点生成"（24px 粗体）
  - 提示文字：
    - "系统将根据教学主题自动生成教学检查点计划"
    - "您可以在下一步查看和调整检查点内容"
  - 信息图标：ℹ️

### 5. 底部操作区
- **布局**：居中对齐
- **内容**：
  - 主按钮："开始教学 →"
  - 尺寸：大按钮 200px × 56px
  - 样式：主按钮绿色主题

## 交互说明

### 学生配置切换
- 点击切换按钮切换学生创建模式
- 切换时保留当前模式的数据

### 手动添加学生
1. 填写学生信息（姓名必填，其他可选）
2. 点击"[+ 添加学生]"按钮
3. 学生添加到列表
4. 表单重置，可继续添加
5. 点击学生旁的删除按钮移除学生

### 随机生成班级
1. 调整班级人数和比例滑块
2. 点击"[🎲 随机生成班级]"按钮
3. 显示生成的学生列表
4. 可调整参数后重新生成

### JSON导入
1. 粘贴 JSON 数据到文本域
2. 点击"[📥 导入数据]"按钮
3. 验证 JSON 格式
4. 成功后显示学生列表
5. 失败显示错误提示

### 表单验证
- 教学主题：必填
- 学生列表：至少 1 名学生
- 验证失败时显示错误提示，阻止提交

### 提交流程
1. 点击"开始教学"按钮
2. 验证所有必填项
3. 验证通过后：
   - 调用后端 API 创建会话
   - 后端自动生成检查点计划
   - 跳转到检查点编辑器页面 `/teacher/checkpoint-editor`
4. 验证失败显示错误提示

## API 接口

```typescript
// 创建教师模式会话
POST /teacher/sessions

Request:
{
  "topic": "Python函数的定义与调用",
  "students": [
    {
      "name": "张三",
      "level": "A",
      "attitude": "positive",
      "learning_ability": 8
    }
  ]
}

Response:
{
  "session_id": 1,
  "checkpoint_plan_id": 1,
  "redirect_to": "/teacher/checkpoint-editor?session_id=1"
}
```

## 设计备注

- 与观察模式配置页面保持一致的布局和交互
- 使用绿色主题区分教师模式
- 省略教学模式选择步骤（教师模式不涉及）
- 强调检查点自动生成特性，引导用户进入编辑器
- 表单验证提示清晰明确
- 保持简洁，避免用户困惑
