## 页面描述

观察模式的核心界面，用户在此实时观察AI教师与学生自动互动的教学过程。页面分为三个区域：顶部状态栏、主内容消息流、右侧边栏（检查点进度 + 学生概览）。

## 组件列表

### 1. 顶部状态栏
- **位置**：页面顶部固定
- **布局**：左中右三栏
- **内容**：
  - 左侧：返回按钮（← 返回）
  - 中间：页面标题"观察模式 - 实时观察"
  - 右侧：状态徽章组
    - 教学模式标签："启发式"（紫色标签）
    - 检查点进度："检查点 2/5"

## 交互说明

### 实时更新
- 页面通过 WebSocket 接收实时消息
- 新消息自动滚动到视图底部
- 检查点状态变更时更新进度列表
- 学生统计数据实时更新

### 消息交互
- 所有消息为只读，不可编辑
- 点击消息可高亮显示（用于聚焦关注）
- 消息支持选择文本复制

### 加载历史消息
- 点击"加载更多"加载更早的消息
- 加载时显示加载动画
- 没有更多消息时按钮禁用

### 检查点进度
- 点击检查点项可查看该检查点的详细信息
- 当前检查点高亮显示

### 学生列表
- 点击学生卡片可查看该学生的详细数据
- 学生卡片按知识掌握度排序（高到低）
- 支持按参与度/发言次数排序切换

## WebSocket 事件

### 接收事件
```javascript
// 新消息
{
  "type": "new_message",
  "data": {
    "sender": "teacher",
    "messageType": "LECTURE",
    "content": "...",
    "timestamp": "..."
  }
}

// 检查点状态变更
{
  "type": "checkpoint_state_change",
  "data": {
    "index": 2,
    "checkpoint": {...},
    "progress": {"current": 2, "total": 5}
  }
}

// 学生数据更新
{
  "type": "student_update",
  "data": {
    "studentId": 1,
    "knowledgeLevel": 0.85,
    "participationRate": 0.75
  }
}
```

## 设计备注

- 页面布局固定，消息流区域可滚动
- 新消息自动滚动到底部，用户手动滚动时暂停自动滚动
- 右侧边栏固定，不随消息滚动
- 检查点状态使用图标和颜色双重编码，便于快速识别
- 学生数据使用进度条直观展示
- 保持消息类型样式一致，便于用户快速识别
