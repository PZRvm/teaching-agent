## 组件列表

### 1. 导航栏
- **位置**：页面顶部
- **内容**：
  - 左侧：返回按钮（← 返回）
  - 中间：页面标题"观察模式 - 会话配置"
- **样式**：
  - 背景：白色半透明
  - 底部边框：2px 手绘虚线

### 2. 步骤 1：教学主题输入
- **布局**：卡片容器，内边距 24px
- **背景**：黄色便签色 `var(--sticky-yellow)`
- **组件**：
  - 标题："步骤 1：教学主题"（24px 粗体）
  - 提示文字："请输入本次教学的主题内容"
  - 输入框：
    - 宽度：100%
    - 占位符："例如：Python变量与数据类型"
    - 手绘边框输入框样式

### 3. 步骤 2：教学模式选择
- **布局**：卡片容器，内边距 24px
- **背景**：黄色便签色
- **组件**：三个教学模式选项按钮

#### 教学模式按钮
- **布局**：横向排列，间距 16px
- **样式**：大按钮 120px × 100px

**灌输式按钮**
- 背景：浅橙色 `var(--sticky-orange)`
- 图标：📢
- 标题："灌输式"（18px 粗体）
- 描述："单向讲授"（14px）
- 选中状态：3px 粗边框 + 阴影加深

**启发式按钮**
- 背景：浅紫色 `var(--sticky-purple)`
- 图标：💡
- 标题："启发式"
- 描述："讲授+提问"
- 选中状态：3px 粗边框 + 阴影加深

**讨论式按钮**
- 背景：浅粉色 `var(--sticky-pink)`
- 图标：💬
- 标题："讨论式"
- 描述："频繁互动"
- 选中状态：3px 粗边框 + 阴影加深

### 4. 步骤 3：学生配置
- **布局**：卡片容器，内边距 24px
- **背景**：黄色便签色
- **组件**：

#### 学生创建方式切换
- 三个切换按钮：
  - "手动添加"（默认选中）
  - "随机生成"
  - "JSON导入"
- 样式：小按钮，选中时蓝色高亮

#### 手动添加模式
- **学生列表**：
  - 显示已添加的学生卡片
  - 每个学生显示：姓名、等级标签(A/B/C)
  - 每个学生旁边有删除按钮

- **添加学生表单**：
  - 姓名：输入框
  - 等级：单选（A-优秀 / B-中等 / C-基础）
  - 态度：单选（积极 / 中立 / 被动）
  - 学习能力：滑块（1-10）
  - 添加按钮："[+ 添加学生]"

#### 随机生成模式
- **配置项**：
  - 班级人数：数字输入框（默认 10）
  - 优秀比例：滑块（10%-50%，默认 20%）
  - 中等比例：滑块（30%-70%，默认 60%）
  - 基础比例：自动计算
  - 生成按钮："[🎲 随机生成班级]"

- **生成结果**：
  - 显示生成的学生列表（只读）
  - 支持"重新生成"

#### JSON导入模式
- **配置项**：
  - 文本域：用于粘贴 JSON 格式的学生数据
  - 占位符：显示 JSON 示例
  - 导入按钮："[📥 导入数据]"
  - 验证提示：显示导入是否成功

### 5. 底部操作区
- **布局**：居中对齐
- **内容**：
  - 主按钮："开始观察 →"
  - 尺寸：大按钮 200px × 56px
  - 样式：主按钮蓝色主题

## 交互说明

### 教学模式选择
- 默认选中"启发式"
- 点击任意模式按钮切换选中状态
- 选中后更新按钮样式（粗边框 + 深阴影）

### 学生配置切换
- 点击切换按钮切换学生创建模式
- 切换时保留当前模式的数据

### 手动添加学生
1. 填写学生信息（姓名必填，其他可选）
2. 点击"[+ 添加学生]"按钮
3. 学生添加到列表
4. 表单重置，可继续添加
5. 点击学生旁的删除按钮移除学生

### 随机生成班级
1. 调整班级人数和比例滑块
2. 点击"[🎲 随机生成班级]"按钮
3. 显示生成的学生列表
4. 可调整参数后重新生成

### JSON导入
1. 粘贴 JSON 数据到文本域
2. 点击"[📥 导入数据]"按钮
3. 验证 JSON 格式
4. 成功后显示学生列表
5. 失败显示错误提示

### 表单验证
- 教学主题：必填
- 学生列表：至少 1 名学生
- 验证失败时显示错误提示，阻止提交

### 提交流交
- 点击"开始观察"按钮
- 验证所有必填项
- 验证通过后跳转到观察页面 `/observation/view`
- 验证失败显示错误提示

## 设计备注

- 三个步骤清晰区分，使用卡片容器
- 每个步骤的标题使用数字标记增强可读性
- 学生配置部分需要根据选择的模式显示不同内容
- 保持表单简洁，避免过多选项导致用户困惑
- 输入框和按钮统一使用手绘风格
