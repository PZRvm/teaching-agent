## 页面描述

教师模式的核心界面，用户（扮演教师角色）在此控制教学流程。页面分为两个区域：左侧消息流显示教学历史，右侧控制面板提供教学操作功能。

## 交互说明

### 实时更新
- 页面通过 WebSocket 接收实时消息
- 新消息自动滚动到视图底部
- 学生数据实时更新

### 教学操作
1. **广播讲授**：
   - 点击"广播讲授"按钮
   - 在输入框输入讲授内容
   - 点击"发送"按钮
   - 消息发送到消息流

2. **向全体提问**：
   - 点击"向全体提问"按钮
   - 在输入框输入问题
   - 点击"发送"按钮
   - 等待学生回答（多个学生可回答）
   - 可选择"结束对话"进入下一个问题

3. **指定学生提问**：
   - 点击"指定学生提问"按钮
   - 在学生选择器中选择目标学生
   - 在输入框输入问题
   - 点击"发送"按钮
   - 等待该学生回答
   - 可选择"结束对话"或继续追问

4. **回复学生**：
   - 当学生提问或回答后，自动显示回复选项
   - 在输入框输入回复内容
   - 点击"发送"按钮

### 对话循环
- 教师提问后，进入对话循环
- 显示当前对话状态（学生回答中 / 教师回复中）
- 可选择"结束对话"按钮结束当前对话
- 结束对话后触发旁听学习机制

### 推进检查点
- 点击"推进检查点"按钮
- 如果有进行中的对话，提示确认结束对话
- 确认后推进到下一个检查点
- 更新检查点卡片内容

### 布置作业
- 点击"布置作业"按钮
- 显示作业输入框（替换内容输入区）
- 输入作业内容
- 点击"发送"布置作业
- 等待学生提交

### 结束教学
- 点击"结束教学"按钮
- 显示确认对话框
- 确认后生成教学报告
- 跳转到报告页面

### 消息交互
- 点击消息可高亮显示
- 消息支持选择文本复制
- 教师消息可编辑（2分钟内）
- 编辑后显示"已编辑"标记

## 设计备注

- 左侧消息流用于显示教学历史，保持上下文
- 右侧控制面板固定，方便教师随时操作
- 教学操作按钮使用不同颜色区分功能
- 当前检查点信息始终可见，帮助教师把握进度
- 学生选择器用于指定学生场景
- 对话循环状态清晰显示，避免教师迷失
- 推进检查点时提供确认，防止误操作
- 教师消息与系统消息使用不同样式区分
- 绿色主题贯穿整个教师模式
